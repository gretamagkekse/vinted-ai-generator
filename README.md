# Vinted AI Generator

**Live Application:** [vinted-frontend-rnqehl2d5a-ew.a.run.app](https://vinted-frontend-rnqehl2d5a-ew.a.run.app)

Vinted AI Generator is an automated listing generator that leverages Google Vertex AI to analyze product images and instantly produce structured, sales-optimized descriptions for Vinted.

---

## Core Features

- **Multi-Image Processing:** Supports batch uploads with automatic, client-side conversion of HEIC files to JPEG to ensure seamless API compatibility.
- **Intelligent Visual Analysis:** Built on the Gemini 2.0 Flash model. The AI autonomously identifies brands, materials, styles (e.g., "Y2K", "Vintage"), colors, and potential clothing defects directly from the provided images.
- **Strict Formatting Constraints:** The system forces the generative model to output a verified 4-paragraph layout:
  1. *Detailed Description* (including at least 3 SEO-optimized hashtags)
  2. *Estimated Original Price* (calculated procedurally based on visual quality indicators if the brand is unknown)
  3. *Condition & Defects* (extracted from the photos or explicitly provided by the user)
  4. *Legal Disclaimer* (standardized private sale clause)
- **Extensive Internationalization (i18n):** Fully supports 14 supported Vinted regions (including EN, FR, DE, ES, IT, PL). Translates outputs and internal JSON structural keys entirely on-the-fly.
- **Interactive Refinement:** Allows users to provide natural language post-processing instructions to edit or refine the generated listing without re-uploading the images.

---

## System Architecture

- **Frontend:** React (powered by Vite)
  - Implements complex state management for multi-image queues and recursive language syncing.
  - Follows a minimalistic, highly objective design system using the Vinted brand color palette.
- **Backend:** Python (FastAPI)
  - Manages asynchronous image packaging and HTTP integrations with Google Cloud Vertex AI.
  - Implements a robust bracket-counting JSON validation parser to automatically sanitize extraneous conversational output often generated by large language models.
- **Cloud Infrastructure:** Google Cloud Run (Serverless)
  - **Dockerized Modules:** Both the API and the Frontend client operate within optimized Alpine Linux containers.
  - **Dynamic Nginx Proxy:** The frontend container utilizes `nginx:alpine` and an `envsubst` initialization script to dynamically generate a secure reverse proxy (`/api/*`). This completely eliminates Cross-Origin Resource Sharing (CORS) security hurdles while preserving deployment flexibility.
  - **Enhanced Payload Capacities:** The Nginx reverse proxy is specifically tuned with a `client_max_body_size` of 32MB to comfortably allow bulk-uploads of high-resolution smartphone imagery.

## Local Development Setup

### Prerequisites
- Node.js (for the React client)
- Python 3.11+ (for the FastAPI backend)
- Google Cloud CLI (`gcloud`) authenticated to a project with Vertex AI enabled

### 1. Initialize the Backend
```bash
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python3 main.py
```
*(The API server will listen on `localhost:8080`)*

### 2. Initialize the Frontend
```bash
cd frontend
npm install
npm run dev
```
*(The React application runs on `localhost:3000` and automatically proxies `/api` traffic to the backend)*

### 3. Serverless Cloud Deployment
The repository includes a root-level bash script (`deploy.sh`) which automates the build process for both containers, validates the deployment context, and provisions the Cloud Run environment.

```bash
chmod +x deploy.sh
./deploy.sh
```
